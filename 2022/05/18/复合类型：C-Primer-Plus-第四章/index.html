数组

float loans[20]；

数组初始化：

内置类型不会初始化为0，如果只初始化一部分，编译器会把剩下的初始化为0，所以建立一个全0数组很简单只需要显式初始化第一个元素为0

int cards[4] = {1,2,3,4};
int cards[] = {1,2,3};  //编译器自动计算长度

//C++11 初始化方法，列表初始化，作为一种通用初始化方式，可用于所有类型
int a[5]{}; // 全0
//列表初始化禁止窄转换
int a[5] = {25,6,3.0} //not allowed

sizeof 数组名返回数组大小

枚举

enum提供了另一种创建符号常量的方式，可以代替const

声明

enum color{red,blue,green};

red blue green 等作为符号常量，对应整数值0~2，这些常量叫做枚举常量

用法：

color c;
c = blue;
// 枚举只有复制运算符没有算数运算符
c++ 不合法

实际上，枚举更常被用来定义相关的符号常量而不是新的类型，如果只打算使用常量不创建枚举类型变量，可以省略枚举类型名称

指针

面向对象编程与面向过程编程的区别在于，OOP强调在运行阶段（而不是编译阶段）进行决策

更加灵活

指针不是整型，虽然计算机通常把地址当做整数来处理，不能将整型赋给指针（要强制类型转换）

new

在程序运行时分配内存

前面我们都将指针初始化为变量的地址，变量是在编译时分配的有名称的内存，指针只是提供了一个别名

指针的真正用武之地在于为运行阶段分配未命名的内存以储存值，这种情况只能通过指针访问内存

malloc 库函数 new 运算符

编译时给数组分配内存被称为静态联编static binding，意味着数组实在编译时加入到程序的，但使用new 时，如果在运行阶段需要数组，则创建它，如果不需要则不创建，还可以在运行时选择数组的长度，称为动态联编dynamic binding

编译器不知道指针是不是指向数组的，要自己维护数组的元素个数，虽然程序确实跟踪了分配的内存量（delete []可以正常释放那么大的空间）但是这个信息不公开，不能用sizeof获取动态数组的字节数

自动存储、静态存储和动态存储

根据用于分配内存的方法，C++有三种管理数据内存的方式，自动存储、静态存储和动态存储（有时也叫自由存储空间或堆）

  1. 自动存储：函数内定义的常规变量使用自动存储空间，称为自动变量automatic variable，是局部变量

    自动变量通常存在栈中，代码块执行时一次压入栈中，离开代码块时相反的顺序释放变量

  2. 静态存储：整个程序执行期间都存在的存储方式，使变量称为静态的方式有两种，一种是在函数外面定义，另一种是声明是使用static

  3. 动态存储：new和delete管理了一个内存池，在C++中称为自由存储空间或堆。该内存池和静态变量、自动变量是分开的。数据的生命周期不受程序或函数的生存时间控制。

    在栈中，自动添加和删除机制使得占用的内存总是连续的，但new和delete使得自由存储区不连续

数组的替代品 vector array

array是c++11新增的，固定长度的数组，使用栈区内存（静态存储区）

vector存在自由存储区
