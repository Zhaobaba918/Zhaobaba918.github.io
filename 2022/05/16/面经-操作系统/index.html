操作系统

进程通信 interprocess communication IPC

信号 signal

signal可以理解为软件中断，打算现在进行应用程序，解决更紧急的事情

操作系统给某个程序发signal

管道

用于把小程序组合起来

一个程序的输出重定向为另一个程序的输入

shell创建两个子进程

把ls的输出从stdout重定向都管道

more的输入stdin重定向到管道

ls more 共享父进程的文件资源buffer

buffer满或者空会阻塞对应的进程

进程间通信有两种基本模式：

消息传递与共享内存（间接通信/直接通信）

消息传递对于交换交少量的数据很有用，因为不需要避免冲突，更容易实现

共享内存允许以最快的速度进行方便的通信

共享内存系统

程序执行过程

  1. 程序员通过编辑器创建源程序（文本文件），ASCII形式保存

  2. 计算机只能执行机器语言，需要编译器把高级语言转换为机器语言，然后把机器语言的集合打包成一个可执行程序，以二进制磁盘文件的形式保存

    编译的过程分为四个阶段 预处理 编译 汇编 链接

    • 预处理：根据#开头的命令修改源代码，比如.h直接插入源代码中
    • 编译：编译器将文本文件hello.i编译成汇编语言文本文件hello.s，汇编语言是非常有用的，因为他为不同的高级语言编译器提供了通用的输出语言
    • 汇编：汇编器将hello.s翻译成机器指令，并打包成可重定位目标程序，将结果保存在目标文件hello.o中，hello.o是二进制文件（汇编语言和机器码等价的，汇编语言是中间语言，可能是为了做优化用）
    • 链接：hello程序调用了printf函数，是每个C编译器都提供的标准库中的函数，printf函数存在于一个名为printf.o的单独预编译文件中，链接器将printf.o合并到hello.o中，得到hello文件，这是一个可执行文件，可以被加载到内存中由系统执行

王道

系统调用：

系统调用system call提供操作系统服务接口，Linux系统调用举例 fork kill mkdir

操作系统运行机制：

内核程序运行在内核态，可以执行内核指令；用户程序运行在用户态，出于安全考虑不能执行特权指令

中断、异常、系统调用

操作系统内核和外部打交道通过中断异常系统调用三种方法

中断：硬件设备对操作系统提出的处理请求

异常：非法指令或其他原因导致当前指令执行失败

系统调用：应用程序主动向操作系统发出服务请求

（以上操作系统指的都是内核态）

广义的中断也可以理解为以上三种方法的总称，硬中断指外设中断，软中断指系统调用和异常

系统调用量很大，占用一个中断编号，再取系统调用表中选择系统调用实现

标准库会封装函数调用

我们在写程序的时候通常不直接使用系统调用，而是把系统调用封装到一个库中，比如C++标准库，应用程序访问库函数间接使用系统调用

其他库：

系统调用时需要切换堆栈，内核态和用户态用不同的堆栈，为了安全

例：时钟中断，表示一个固定的时间片已到，让处理机处理计时、启动定时任务等

系统启动过程

内存分为：RAM随机访问存储和ROM只读存储

ROM加电之后还会有以前写的内容，系统的初始化代码从这里开始

进程

进程由程序端，相关数据段和PCB组成，进程是程序的一次执行

进程状态

创建态，就绪态，运行态，阻塞态，结束态

进程通信

  1. 共享内存+信号量：最快的ipc，需要进行同步，通常与信号量结合使用
  2. 消息队列：消息的链表，存在内核当中，用户进程可以添加消息也可以读取消息，独立于接收进程和发送进程，
  3. 管道通信：半双工的，同一时刻只能单向传递，可以看成是共享存储的优化，因为可以同时读写，不会因为互斥而被阻塞。管道存在于内存中
  4. socket通信：不同主机的进程通过操作系统提供的socket接口进行网络通信，分为TCPsocket和UDPsocket

进程间通信IPC需要进程同步和互斥手段辅助，以保证数据的一致性，而线程间可以直接读写进程数据段（如全局变量）来进行通信？？？

处理机调度

作业调度：将作业从外存加载到内存，创建进程

中级调度：又称内存调度，提高内存利用率和吞吐量，把暂时不能运行的内存调到外存，进程进入挂起态

进程调度

进程调度

  1. 抢占式
  2. 非抢占式

调度算法

有的调度算法适用于作业调度，有的适用于进程调度，有的都适用

  1. 先来先服务FCFS：可用于作业调度，进程调度，非抢占式
  2. 短作业有限SJF：非抢占式，平均等待时间平均周转时间最短
  3. 优先级调度：作业调度或进程调度，抢占或非抢占
  4. 高响应比优先：非抢占，FCFS和SJF的综合，响应比=(等待时间+要求服务时间)/要求服务时间
  5. 时间片轮转算法：
  6. 多级反馈队列：抢占式，时间片轮转和优先级调度的综合

进程同步

临界资源：一次只允许一个进程访问的资源

同步：同步又称直接制约关系，是指完成某种任务而建立的两个或多个进程，需要有先后执行的关系，制约关系源于进程间的合作

互斥：互斥又称间接制约关系，访问临界资源而产生的制约

为了禁止连个进程同时进入临界区，同步机制应遵循以下准则：

  1. 空闲让进
  2. 忙则等待
  3. 有限等待
  4. 让权等待（可选）

实现临界区互斥的方法

软件方法

依赖条件少，但是实现复杂

  1. 单标志法 （turn=0 turn=1只能两个进程，不满足有空让进）
  2. 双标志先检查法 （可能同时进入临界区）
  3. 双标志后检查法 （可能谁也进不了，饥饿现象）
  4. Peterson算法 （双标志+turn）

软件方法只能实现两个进程互斥，并且设计复杂，只能通过模拟来理解

硬件实现

利用特殊的指令，如对一个字中的内容检测和修正，对两个字的内容交换

  1. 关中断：CPU只在发生中断时引起进程切换（适用单处理机）

  2. 硬件指令：TestAndSet

    Swap

    通过这两个原子操作可以实现锁，可以是忙等待或非忙等待

硬件方法优点：适用于任意数目的进程，容易验证正确性

缺点：不能实现让权等待；每次随机选一个进程进入临界区，有的进程可能一直进不去造成饥饿

信号量

用的最多

信号量和管程都属于软件进程互斥的软件实现方法

信号量只能被两个标准的原语P V访问，原语具有原子性，通常硬件实现

死锁

死锁出现的必要条件：

  1. 互斥
  2. 持有并等待
  3. 非抢占
  4. 循环等待

死锁处理方法

预防

确保系统不会陷入死锁，使系统任何时刻不满足死锁的必要条件

针对四个条件来说

避免

使用前进行判断，只允许不会出现死锁的进程请求资源

银行家算法

  1. 定义了系统安全状态：一个进程串行执行的序列
  2. 接下来试探性的扫描正在申请资源的进程，如果分配资源后系统处于安全状态就分配，否则不分配
  3. 这个过程中要维护一些数据结构，包括各种资源额剩余数量、各个进程资源最大需求量，当前持有量，当前需求量（通过前两个矩阵可以算出来）
  4. 复杂度m*n^2 m是资源数，n是进程数
死锁检测和恢复
由应用进程处理死锁

通常操作系统忽略死锁，这是大多数操作系统的做法

内存

内存的最小访问单位是字节

计算机系统是32位的总线，一次读写可以从内存中读或写32位

地址空间和地址生成

物理地址与存储单元一一对应，写程序需要逻辑地址，这样不用在写程序时将地址写死，程序容易移植

逻辑地址：CPU运行时进程看到的地址空间

非连续内存分配

非连续内存分配根据分区的大小是否固定分为页式存储管理和段式存储管理

页式存储管理根据运行作业时是否要把作业的所有页面都装入内存才能运行分为基本分页存储管理和请求分页存储管理

基本分页存储管理

分页的思想：把主存空间划分为大小相等且固定的块，块相对较小，作为主存的基本单位。每个进程也按块进行划分，进程执行时以块为单位逐个申请主存中的空间

概念：

  1. 页面和页面大小：进程中的块成为页page，内存中的块称为叶框或页帧page frame，
