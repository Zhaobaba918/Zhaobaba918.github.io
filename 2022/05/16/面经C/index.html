内存泄漏

内存泄漏 memory leak：指程序申请内存后无法释放已经申请的内存。内存泄漏堆积以后可能导致内存溢出OOM out of memory

在C++里就是new 完了不delete，结果new 的内存自己也不能访问了（没有指向他的指针），系统也不能再次将这块空间分配

内存泄漏常见的三种情况：

  1. 指针重新赋值

  2. 错误的内存释放（释放结构话内存（类），应先遍历释放自内存）

    free(p->np);
free(p);

  3. 返回值的不正确处理

  4. new 后 忘了delete

避免内存泄漏：和原因相对

栈溢出

原因：

  1. 局部数组过大
  2. 递归层数太深
  3. 指针越界

解决方法：增大栈空间，或改用动态分配，用堆空间

alloca malloc calloc realloc

void* malloc(unsigned size);
void* realloc(void* ptr, unsigned newsize); 
void* calloc(size_t numElements, size_t sizeOfElement);

malloc：申请堆内存，没有初始化，一般和memset连用

calloc：与malloc相似,参数sizeOfElement为申请地址的单位元素长度,numElements为元素个数，即在内存中申请numElements*sizeOfElement字节大小的连续地址空间，并初始为0（每一位初始为0）

realloc：对malloc申请的内存大小调整，如果原有空间后面有足够的空余空间直接用，没有的话就重新找一片足够大的连续空间，再深拷贝。以前的指针可能会失效

alloca：申请栈内存，无需释放。alloca函数分配的内存不需要手动释放，和普通的栈上对象的处理一样：超出作用域自动回收内存。alloca函数并不被推荐使用，因为它不安全。
