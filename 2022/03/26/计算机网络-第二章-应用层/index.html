第二章 应用层

CS模式：扩展性差（服务器端） 可靠性差（都依赖服务器）

P2P架构：节点既可以做服务器也可以做客户端 目录查询是集中式 文件获取是分布式

TCP socket是一个整数（类似连接id）代表会话关系 包括本机ip、端口号，目的ip、端口号 是一个本地标识 便于管理 发送信息的接口参数变少

UDP socket是二元组 保存本机ip udp端口 发送报文时要制定对方ip 和udp端口

应用层协议规范了格式和语法

Web 和 http

Web页：由一些对象组成，对象可以是HTML文件，JPG图像、声音文件等

Web页含有一个基本的HTML文件，该文件包含若干对象的引用（URL）

任何对象可以采用URL定位

URL格式：协议，用户名，口令字，端口等

支持匿名访问不用用户名和口令

http

http 跑在 TCP上

服务器在某个ip的80端口上运行等待客户端的连接建立请求 有一个socket在等待来自用户端的请求（守候socket）

建立连接（建立socket）

然后就可以发http报文了

http协议无状态 不保存之前通信的历史

http1.1 默认持久连接，发送完一个 数据不断开连接还可以发别的

时间相应模型

往返时间RTT round trip time：一个小的分组从客户端到服务器再返回客户端的时间（发送时间忽略）

非持久性连接发送一个对象至少两个RTT 一个RTT是建立简介时间

持久性连接分为流水线方式和非流水线方式（是不是一个对象回来再请求下一个对象）

HTTP请求报文

请求报文和响应报文是ASCII编码，可读的

GET POST HEAD（只请求HTML的头部，搜索引擎建立索引用）

PUT：将尸体主体重的文件上载到URL字段规定的文件

DELETE：删除URL字段规定的文件

响应报文

http运行在TCP上，TCP交字节流，采用TCP的协议自己维护报文的界限

HTTP

无状态协议

好处：简单

cookie

客户端第一次发送请求不带cookie

服务端返回一个cookie，在头部携带，同时cookie保存在服务器

客户端将cookie存在本地

Web缓存

缓存通常是由ISP安装（大学、公司、社区）

好处：降低客户端请求响应时间；减少一个机构内与Internet接入链路上的流量；减少ICP的流量，使较弱的ICP也能够提供有效的内容

代理服务器如何与ICP同步

如果ICP的东西在某一时间内没有修改，不用返回内容主体，返回一个头部就行

FTP

早期分享文件使用

ftp服务器守候在21号端口，等待用户端建立连接

控制信息传输21号端口 数据传输20号端口

底层TCP

用户名密码明文传输

ftp有状态

电子邮件

三个主要组成部分：

  1. 用户代理 outlook qq Foxmail
  2. 邮件服务器 守候在25号端口
  3. 协议

用户代理配置好邮件服务器的ip和端口

发送的时候用户代理发给邮件服务器，邮件服务器从队列中取出邮件发给目标邮件服务器（SMTP协议），目标邮件服务器将邮件存着，用户通过用户代理拉去邮件（很多拉取协议）

SMTP持久连接

报文必须是7位ASCII码

base64编码：将若干个不在ASCII码内的字节编码为在ASCII码内的字节

DNS

domain name system

可以做负载均衡

DNS域名结构

Internet被划分为几百个顶级域

通用的：.com .edu .gov

国家的：.cn .us

每个域下面可划分为若干个子域

叶节点是主机

有13个根域名服务器

域名怎么划分由顶级域决定（.jp .cn具体怎么划分自己说了算）

每个zone内部安排一个权威名字服务器，维护这个域的所有名字到ip的权威信息

TLD顶级域服务器：负责顶级域名.com org net和所有国家级顶级域名.cn .jp

上层域有个指针指向下级DNS服务器

DNS服务器存储的资源记录：域名，ttl（权威域名服务器是无限大，其他服务器为了一致性会设置过期时间，与缓存一个道理）

local dns server要么手工配，要么DHCP自动配

local dns server一般在同一子网内部，因为快

查询过程：

  1. 主机发起查询，查询被送到本地DNS服务器，

根服务器的内容通常都在本地服务器中缓存着，使得根服务器不经常被访问

新增一个域：到机构注册域名（.com为例），需要提供权威DNS服务器的名字和ip地址，登记机构在com TLD服务器中插入两条记录，一条记录记录该域的dns服务器名，一条记录dns服务器地址

主机名别名的作用

一个主机可以提供不同的服务 比如www和mail这样有两个名字便于用户记忆和区分 实际上这两个别名指向一个真正的主机名

P2P应用

流量是分布的

大量peer节点相互服务 很容扩展业务规模

CS 模式

CDN

杀手应用：占大部分带宽的服务 如视频

视频压缩后在网络传输，前提是视频在时间或空间上有冗余度，压缩成小码率

多媒体流化服务：DASH 视频流，一边传一边播

Dynamic Adaptive Streaming over HTTP 动态自适应

服务器将视频分割成多个块，用不同码率编码，告示文件manifest file提供不同块的url

客户端点播视频线下载manifest file，根据带宽选择每一块的码率

CDN内容分发网络：Content distribution networks 内容靠近用户

原理：CDN全球部署很多缓存节点，数据预先存在CDN节点，视频运营商买CDN服务，用户点播的时候被动态重定向到合适的节点，离用户比较近跳数比较少有利于提供服务
